Este repositório apresenta a implementação do padrão de projeto Flyweight aplicada a um cenário de gerenciamento de passagens aéreas. O objetivo principal do padrão é otimizar a utilização de memória por meio do compartilhamento de objetos que possuem características intrínsecas idênticas, evitando a criação repetitiva e desnecessária dessas estruturas.

No sistema desenvolvido, a classe Voo representa as informações estáticas e compartilháveis, como número, origem e destino. Para garantir que essas instâncias não sejam duplicadas, foi criada a VooFactory, responsável por centralizar e reutilizar objetos existentes sempre que uma nova passagem é solicitada para o mesmo voo. Dessa forma, diversas passagens podem referenciar um único objeto de voo sem comprometer a integridade dos dados.

A classe Passagem representa os dados extrínsecos da operação, como passageiro, assento e preço, que variam em cada compra. Já a classe CompanhiaAerea atua como cliente do padrão Flyweight, utilizando a fábrica para obter os voos e criar as passagens correspondentes. Esse desacoplamento permite a manutenção simples e eficiente das entidades, reduzindo o custo de instanciamento e permitindo que o sistema escale com melhor desempenho.

A implementação evidencia como o Flyweight se torna essencial em contextos que envolvem grande volume de objetos semelhantes, como sistemas de reservas, catálogos, jogos e simulações. Ao separar os atributos compartilháveis dos específicos, o padrão proporciona economia significativa de recursos, mantendo o sistema leve, organizado e pronto para expansões futuras.
